## Quick orientation

- Current version: **V5.1** — tier-gated moderator handbook + multi-tier notification bells (grey/blue/gold/purple)
- Stack: Next.js 15 App Router · TypeScript · Tailwind v4 · Drizzle + Neon · iron-session · Steam OpenID
- Hard constraint: everything must remain on **free tiers** — no paid APIs, no metered per-item costs
- Transactions are **not available** (neon-http driver) — writes are sequential, partial state is accepted

- **V5.1 — moderator handbook + tier bells.** Two polish additions after the public launch landed. (1) `/admin/guide` renders a tier-gated moderator handbook — a regular mod sees only the moderator section, an elite sees mod + elite, the creator sees everything. Each section walks through every admin route + server action that tier can use, plus a shared House Rules section with warn-before-mute / paper-trail / no-self-moderation guidance. (2) Notifications now carry a `tier` field (`user` / `moderator` / `elite_moderator` / `creator`) and the header renders up to four coloured bells — grey for personal, blue for moderator, gold for elite, purple for creator — each with its own unread count. `/me/notifications` gets tabs for each tier the viewer has access to; mark-all-read is scoped to the active tab so you don't lose unread state on other tiers when you check one. Seed events wired so bells aren't dead on arrival: a new user report fans out to every mod+; a creation archive fans to every elite+ (excluding the actor); a new feature suggestion fans to the creator. `lib/db/notifications.ts` gains `broadcastToRole({ minRole, tier, excludeUserId })` — skips banned users and the actor. Migration `0013_normal_fantastic_four.sql` adds the `tier` column with a `user` default so existing rows keep their current behaviour.
- **V5.0 — public launch hardening.** Multi-agent audit + fix pass before opening the site to the public. Security: `requireMod()` added to eight admin server actions that relied only on the layout gate (previously any signed-in user could approve/reject items via a direct server-action call); open redirects on `/auth/steam/login`, `/auth/steam/return`, and `/api/prefs/rating-mode` closed by rejecting `//`-prefixed paths; cron routes drop the `?secret=` query branch (header-only); free-text admin fields clamped (ban/mute 300, warn/resolver 500). Privacy: moderation status + `steamCreatedAt` on `/profile/[steamid]` gated to self + mods via new `canSeeModInfo` helper; `VoteHistory` section now self-only; `ReportBadge` no longer leaks reporter custom text or resolver identity — shows only a canonical reason label; `profileUrl` scheme-checked before rendering; `/me/*`, `/verify`, `/profile/[steamid]`, `/suggestions/new`, `/submit` all `noindex`; robots.txt disallow extended. SEO: `metadataBase`, default OG + Twitter on root layout, per-page metadata + canonicals on home / `/new` / `/[kind]` / `/search` / `/suggestions` / `/submit` / `/creation/[id]`; favicon (`app/icon.png`, `app/apple-icon.png`) wired from the existing logo. Rate limits: `voteCreation` / `voteTag` / `toggleFavourite` / `voteSuggestion` = 30 writes per 60 s per user (in-memory); `suggestTag` = 10/hour; `reportCreation` = 1 per creation per 24 h and 5/day total; `submitCreation` pre-check moved above the duplicate check and capped at 5/day; `submitSuggestion` +10/day cap; `recordCreationView` deduped to 1 write per user per creation per hour inline in the UPSERT; `?page=` clamped to ≤200 on listing pages. Error handling: `app/error.tsx` rewritten as a friendly card (ref digest only; dev-mode shows stack); new `app/global-error.tsx` for root layout failures; `app/not-found.tsx` gains a search CTA. Accessibility: mobile drawer focus trap + Escape + `aria-controls`; vote arrows, vote panel, rating toggle, search filters, tag autocomplete, comment delete, submit forms, suggestion form all got proper ARIA (`aria-pressed`, `aria-label`, `role="status"`/`aria-live`, combobox listbox pattern); text colour contrast bumped on the moderation section. Legal: `/terms` + `/privacy` pages with honest, site-specific copy; footer now links Terms / Privacy / GitHub; sitemap lists both. Captcha `/verify` page gains `aria-live` flash + timer cleanup. No new dependencies.
- **V4.17:** Two-part release. (1) Profile pages (`/profile/[steamid]`) now render the three sections the old placeholder promised — "Submitted to the site", "Favourites", "Vote history" (with ↑/↓ badges). Each section hides when the user has nothing in it, so new accounts stay compact. (2) Captcha hardening after a red-team pass. The old `bot_verified=1` cookie was a plain unsigned string — `curl -H "Cookie: bot_verified=1"` walked straight past the captcha. Now sealed via iron-session (`SESSION_SECRET`-signed). Images moved out of `public/` to `lib/captcha/images/` with opaque numeric names (`01.jpg` … `25.jpg`) so there's no guessable URL like `/captcha/Woc1.jpg`; `next.config.ts` `outputFileTracingIncludes` bundles them with the proxy route on Vercel. The image proxy now also rejects anything that isn't `\d+\.jpg` as defence in depth against session-poisoned traversal. Remaining known gap: a repo cloner can still build a perceptual-hash cheat table from the 25 binaries. Closing that needs either private image storage or per-request `sharp`-based jittering — see `docs/captcha.md` threat model.
- **V4.16:** `/search` now runs on a real Postgres full-text index. `creations.search_vector` is a generated-stored tsvector over `title + description_clean`, backed by a GIN index; queries use `websearch_to_tsquery('english', q)` so users get phrase quotes / `OR` / `-negation` for free. A new `relevance` sort mode ranks with `ts_rank_cd` and becomes the default whenever `q` is present. The old ILIKE `%q%` path is gone — stems now match (`cannon`/`cannons`) but substrings don't (`cann` no longer matches `cannon`); revisit with `:*` prefix if that bites. Migration `0012_overrated_alex_power.sql` adds the column + index; no ingest-pipeline change needed because Postgres maintains the column itself.
- **V4.15:** Rating-mode toggle (Steam/Site/Both) stored as cookie, applied across card + detail views. Fixed the "everything is 3 stars" bug — `StarRating` now computes raw `up/(up+down)` instead of trusting Steam's Wilson-smoothed `vote_score` (which regressed low-sample ratings toward 0.5); min-votes threshold raised from 5 to 10. Added `site-rating` / `site-least-rating` sort modes (5-vote floor). Queue reject action accepts an optional reason that surfaces in the submitter's rejection notification. Tightened appid check so a missing `consumer_appid` no longer slips through. Per-notification mark-read via `/api/notifications/[id]/click`. Rebuilt mobile header: sticky compact bar with logo + submit/heart/bell/avatar icons, hamburger opens a right-side drawer (portaled to `document.body` at `z-[100]`) with nav links, rating toggle, deep links (submissions, admin triage), and sign-out.
