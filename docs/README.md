## Quick orientation

- Current version: **V5.6** — `/settings` hub + gear icon in header, light-theme readability pass (accent shades remapped, washed-out alpha bumped)
- Stack: Next.js 15 App Router · TypeScript · Tailwind v4 · Drizzle + Neon · iron-session · Steam OpenID
- Hard constraint: everything must remain on **free tiers** — no paid APIs, no metered per-item costs
- Transactions are **not available** (neon-http driver) — writes are sequential, partial state is accepted

- **V5.6 — settings hub + light-theme readability pass.** Two discoverability / polish items. (1) New **`/settings` page** consolidates theme picker, ratings toggle, quick links to profile / favourites / submissions / notifications, help, and sign-out. Linked from a gear icon in the header next to the tier bells, the footer, and the top of the mobile drawer — so users can find preferences without hunting through the drawer. (2) **Default-theme readability pass.** The codebase was originally written against a dark background, so every `text-sky-300`, `text-emerald-200`, `text-red-300` etc. is a pale shade designed for that context — on the new light default they washed out to near-invisible. Added a block of CSS overrides scoped to `html[data-theme="default"]` that remap each 100/200/300/400 accent shade to a readable 700-ish replacement (sky→#0369a1, emerald→#047857, amber→#b45309, purple→#6d28d9, red→#b91c1c). Also bumped the alpha floor on `text-foreground/20-40` so muted labels stay visible. Dark-family themes (V1, Workshop, Contrast) keep the original pale shades — overrides don't apply to them.
- **V5.5 — three bug fixes from the first day of real usage.** (1) **Submit gate was rejecting every legitimate SM item.** The tightened V4.15 check looked at `consumer_appid === 387990`, but the ISteamRemoteStorage endpoint we use for `/submit` returns `creator_app_id` (note underscores), not `consumer_appid` — so the field was always undefined and `undefined !== 387990` tripped. Fixed to read `creator_app_id` (with `consumer_appid` as fallback) against the set **{387990, 588870}** — both legit Scrap Mechanic Workshop appids. Kind-tag fallback added for the rare case where Steam returns neither appid, so we don't reject entirely on a missing field. (2) **Age gate leaked through for private Steam profiles.** Fresh accounts whose Steam profile hides `timecreated` returned `steamCreatedAt = null`, and the old check (`if (user.steamCreatedAt && ...)`) silently skipped the gate — anyone with a locked Steam profile got unlimited submit/suggest/vote access. Now the null case is **fail-closed**: block unless `bypassAgeGate` is explicitly set for that user. (3) **Contributor-scrape catch-up on the refresh cron.** The initial backfill lost ~60% of rows to transient Steam hiccups and stored empty `creators` arrays. Added a one-retry + 500ms backoff to `fetchWorkshopContributors`, and the weekly refresh cron now takes a second pass through up to 50 empty-creator rows per run (`refreshMissingCreators`) — so the catalog self-heals over the weeks. Re-ran the backfill once more to pick up the majority of previously-failed items.
- **V5.4 — light-by-default + user-configurable theme.** Four requests from the Reddit/ideas-board feedback batch. (1) **Default theme flipped** from black+amber to a warm white+orange palette. The previous dark look stays available as a named option called **V1** — existing users can flip back instantly from the theme pill. Under the hood all ~600 hardcoded `text-white` / `bg-white` / `border-white` utility classes in app + components were rewritten to use theme tokens (`text-foreground` / `bg-foreground` / `border-foreground`) so every theme switch lights up the whole site coherently. The sticky header + drawer + captcha zoom overlay were re-checked to keep their always-dark contexts readable. (2) **Custom theme editor** at `/settings/theme` — five colour inputs (background, foreground, card, accent, border), saved to a hex-validated `smse_theme_custom` cookie, injected as an inline `<style>` overriding `html[data-theme="custom"]` variables. A **Reset to default** button is always there as a fail-safe if the user paints themselves into a one-colour corner. (3) **Creator inline tag-force** on `/creation/[id]` — a new emerald "✓" button next to every unconfirmed tag chip (creator-only) promotes that tag to `confirmed=true` immediately, bypassing the +3 vote threshold. New `confirmCreationTag` server action (mod+) backs it. (4) **Mobile admin horizontal scroll** — wrapped the `/admin/users` table in `overflow-x-auto` with a `min-w` floor so the ban/mute/hard-ban buttons are reachable by swiping sideways instead of rotating the phone.
- **V5.3 — post-launch feedback polish.** Three Reddit-sourced fixes. (1) **Multi-creator bug.** Steam's API only returns a single `creator` per workshop item, but many SM workshop items are collaborations listing 2–10 contributors in the page sidebar. Added `creations.creators jsonb` (migration 0015) and a new `fetchWorkshopContributors(apiKey, id)` helper that scrapes the rendered Workshop HTML's `creatorsBlock`, resolves vanity URLs via `ResolveVanityURL`, and returns `[{steamid, name}]`. Wired into every creation-insert path (ingest cron, `/submit`, `/admin/add`) and into the creation detail page's byline — multi-creator items now render as "by Corvos, MelOnE & wingcomstriker405". (2) **Mobile captcha.** Switched from `object-cover` (cropping the sides off of 3:1 panoramic screenshots at 280px tall) to `object-contain` with `max-h-[60vh]` so the full image is visible on every viewport, letterboxed on mobile. Also made the "Tap to zoom" pill bigger and more prominent on mobile — larger icon, bolder text, stronger background. (3) **Search tile in the mobile drawer.** Search now sits at the top of the drawer as an accent-coloured tile with a magnifier icon ("Search the workshop") instead of being buried seventh in the nav list.
- **V5.2 — trusted-user age bypass + colour themes.** Two community-driven additions after the Reddit launch. (1) New `users.bypassAgeGate` column (migration 0014) lets the creator flip individual users past the 7-day Steam account-age gate from `/admin/users` — useful for trusted community members whose Steam account is fresh. `requireVotingUser` and `requireActiveUser` honour the flag; the gate is still the default for everyone else. (2) Three user-selectable colour themes (`default`, `workshop`, `contrast`) stored in the `smse_theme` cookie. - **V5.1 — moderator handbook + tier bells.** Two polish additions after the public launch landed. (1) `/admin/guide` renders a tier-gated moderator handbook — a regular mod sees only the moderator section, an elite sees mod + elite, the creator sees everything. (2) Notifications now carry a `tier` field (`user` / `moderator` / `elite_moderator` / `creator`) and the header renders up to four coloured bells — grey for personal, blue for moderator, gold for elite, purple for creator — each with its own unread count. `/me/notifications` gets tabs for each tier the viewer has access to; mark-all-read is scoped to the active tab so you don't lose unread state on other tiers when you check one. - **V4.17:** Two-part release. (1) Profile pages (`/profile/[steamid]`) now render the three sections the old placeholder promised — "Submitted to the site", "Favourites", "Vote history" (with ↑/↓ badges). Each section hides when the user has nothing in it, so new accounts stay compact. (2) Captcha hardening after a red-team pass. The old `bot_verified=1` cookie was a plain unsigned string — `curl -H "Cookie: bot_verified=1"` walked straight past the captcha. Now sealed via iron-session (`SESSION_SECRET`-signed). Images moved out of `public/` to `lib/captcha/images/` with opaque numeric names (`01.jpg` … `25.jpg`) so there's no guessable URL like `/captcha/Woc1.jpg`; `next.config.ts` `outputFileTracingIncludes` bundles them with the proxy route on Vercel. The image proxy now also rejects anything that isn't `\d+\.jpg` as defence in depth against session-poisoned traversal. Remaining known gap: a repo cloner can still build a perceptual-hash cheat table from the 25 binaries. Closing that needs either private image storage or per-request `sharp`-based jittering — see `docs/captcha.md` threat model.
- **V4.16:** `/search` now runs on a real Postgres full-text index. `creations.search_vector` is a generated-stored tsvector over `title + description_clean`, backed by a GIN index; queries use `websearch_to_tsquery('english', q)` so users get phrase quotes / `OR` / `-negation` for free. A new `relevance` sort mode ranks with `ts_rank_cd` and becomes the default whenever `q` is present. The old ILIKE `%q%` path is gone — stems now match (`cannon`/`cannons`) but substrings don't (`cann` no longer matches `cannon`); revisit with `:*` prefix if that bites. Migration `0012_overrated_alex_power.sql` adds the column + index; no ingest-pipeline change needed because Postgres maintains the column itself.
- **V4.15:** Rating-mode toggle (Steam/Site/Both) stored as cookie, applied across card + detail views. Fixed the "everything is 3 stars" bug — `StarRating` now computes raw `up/(up+down)` instead of trusting Steam's Wilson-smoothed `vote_score` (which regressed low-sample ratings toward 0.5); min-votes threshold raised from 5 to 10. Added `site-rating` / `site-least-rating` sort modes (5-vote floor). Queue reject action accepts an optional reason that surfaces in the submitter's rejection notification. Tightened appid check so a missing `consumer_appid` no longer slips through. Per-notification mark-read via `/api/notifications/[id]/click`. Rebuilt mobile header: sticky compact bar with logo + submit/heart/bell/avatar icons, hamburger opens a right-side drawer (portaled to `document.body` at `z-[100]`) with nav links, rating toggle, deep links (submissions, admin triage), and sign-out.
